# Архитектура DSP Analytics Platform

## Обзор проекта

**DSP Analytics Platform** - это AI-ядро аналитической платформы для анализа данных DSP (Demand-Side Platform). Проект реализует многослойную архитектуру для обработки, анализа и визуализации рекламных данных с автоматической генерацией инсайтов и рекомендаций.

## Структура проекта

```
qbr/
├── config.py              # Конфигурация и константы
├── data_layer.py          # Слой загрузки данных
├── processing_layer.py    # Слой обработки и агрегации
├── ai_layer.py            # AI-слой генерации инсайтов
├── visualization.py       # Слой визуализации
├── report_generator.py    # Генератор отчетов
├── main.py                # Точка входа
├── requirements.txt       # Зависимости
└── README.md              # Документация
```

## Архитектурные слои

### 1. DATA LAYER (`data_layer.py`)

**Ответственность:**
- Загрузка данных из различных источников (API, CSV, Excel)
- Нормализация типов данных
- Преобразование в единый формат DataFrame

**Классы:**
- `DSPClient` - клиент для работы с DSP API
- `DataLoader` - универсальный загрузчик данных

**Принципы:**
- Инкапсуляция источников данных
- Единый интерфейс для всех типов данных
- Автоматическая типизация метрик

### 2. PROCESSING LAYER (`processing_layer.py`)

**Ответственность:**
- Расчет производных метрик (CTR, CPC, CPM, etc.)
- Агрегация данных по различным измерениям
- Фильтрация статистически значимых данных
- Выполнение аналитических запросов

**Классы:**
- `DataProcessor` - обработка и расчет метрик
- `AnalysisEngine` - движок для различных типов анализа

**Методы анализа:**
- `compare_advertisers()` - сравнение рекламодателей
- `compare_products()` - сравнение продуктов
- `media_split()` - медиасплит по измерениям
- `dynamics()` - анализ динамики
- `dmp_analysis()` - анализ DMP эффективности
- `frequency_analysis()` - анализ влияния frequency
- `top_performers()` - топ по метрикам
- `anomalies()` - поиск аномалий

**Принципы:**
- Взвешенные средние для корректной агрегации
- Поддержка группировки и общих итогов
- Автоматический пересчет производных метрик после агрегации

### 3. AI LAYER (`ai_layer.py`)

**Ответственность:**
- Автоматическая генерация инсайтов
- Создание рекомендаций
- Формулирование гипотез для тестирования
- Сравнение с бенчмарками

**Классы:**
- `InsightGenerator` - генератор AI-инсайтов

**Методы генерации:**
- `_analyze_advertisers()` - анализ рекламодателей
- `_analyze_products()` - анализ продуктов
- `_analyze_media_split()` - анализ медиасплита
- `_analyze_dynamics()` - анализ трендов
- `_analyze_dmp()` - анализ DMP
- `_analyze_frequency()` - анализ частоты
- `_analyze_conversions()` - анализ конверсий
- `_generate_tldr()` - генерация краткой сводки

**Принципы:**
- Контекстные инсайты на основе данных
- Автоматическое сравнение с бенчмарками
- Приоритезация проблем и возможностей

### 4. VISUALIZATION LAYER (`visualization.py`)

**Ответственность:**
- Генерация интерактивных графиков
- Визуализация результатов анализа
- Экспорт в различные форматы (HTML, PNG, PDF)

**Классы:**
- `ChartGenerator` - генератор графиков

**Типы графиков:**
- `advertiser_comparison()` - сравнение рекламодателей
- `media_split_pie()` - pie chart медиасплита
- `dynamics_line()` - линейные графики динамики
- `frequency_scatter()` - scatter plot frequency vs CTR
- `format_performance()` - performance по форматам
- `dmp_comparison()` - сравнение DMP

**Технологии:**
- Plotly для интерактивных графиков
- Поддержка экспорта в HTML/PNG/PDF

### 5. OUTPUT LAYER (`report_generator.py`)

**Ответственность:**
- Генерация отчетов в различных форматах
- Структурирование результатов
- Подготовка данных для презентаций

**Классы:**
- `ReportGenerator` - генератор отчетов

**Форматы вывода:**
- Markdown отчеты
- JSON для программного использования
- Структура для Google Slides (будущая функциональность)

### 6. CONFIGURATION (`config.py`)

**Ответственность:**
- Централизованная конфигурация
- Определение метрик и измерений
- Бенчмарки и пороги

**Константы:**
- `CORE_METRICS` - основные метрики
- `CONVERSION_METRICS` - метрики конверсий
- `DIMENSIONS` - доступные измерения
- `MIN_IMPRESSIONS` - порог статистической значимости
- Бенчмарки (CTR, VTR, Viewability)

### 7. ENTRY POINT (`main.py`)

**Ответственность:**
- Оркестрация всех слоев
- CLI интерфейс
- Демо режим
- Управление потоком выполнения

**Режимы работы:**
- `demo` - генерация синтетических данных и полный анализ
- `api` - загрузка из DSP API
- `csv/excel` - анализ локальных файлов

## Принципы разработки

### 1. Чистая архитектура
- Каждый слой имеет четко определенную ответственность
- Зависимости направлены от внешних слоев к внутренним
- Бизнес-логика изолирована от источников данных

### 2. Минимализм
- Нет избыточных абстракций
- Простые и понятные интерфейсы
- Минимум зависимостей

### 3. Единственная ответственность
- Каждый класс решает одну задачу
- Методы выполняют одну функцию
- Четкое разделение concerns

### 4. Extensibility
- Легко добавить новые источники данных
- Простое расширение метрик
- Модульная структура инсайтов

### 5. Корректность
- Правильные расчеты взвешенных средних
- Обработка граничных случаев (пустые данные, division by zero)
- Типизация и валидация данных

## Взаимодействие модулей

### Загрузка данных
```python
loader = DataLoader()
df = loader.load_from_csv('data.csv')  # или load_from_api() / load_from_excel()
```

### Обработка
```python
processor = DataProcessor()
df = processor.calculate_derived_metrics(df)
df = processor.filter_significant(df)

engine = AnalysisEngine(df)
advertisers = engine.compare_advertisers()
dynamics = engine.dynamics('M')
```

### AI анализ
```python
insight_gen = InsightGenerator(df)
results = insight_gen.generate_full_analysis()
# results содержит все инсайты, рекомендации и гипотезы
```

### Визуализация
```python
chart_gen = ChartGenerator()
charts = chart_gen.generate_all_charts(results, output_dir='./charts')
```

### Отчеты
```python
report_gen = ReportGenerator(results)
report_gen.save_markdown('report.md')
report_gen.save_json('results.json')
```

## Расширение функциональности

### Добавление нового источника данных
1. Добавить метод в `DataLoader`
2. Обеспечить возврат DataFrame с нужными колонками
3. Не требуется изменение других слоев

### Добавление новой метрики
1. Добавить в `config.py` в соответствующий список
2. При необходимости добавить расчет в `DataProcessor.calculate_derived_metrics()`
3. Метрика автоматически будет агрегироваться

### Добавление нового типа анализа
1. Добавить метод в `AnalysisEngine`
2. Добавить метод анализа в `InsightGenerator`
3. При необходимости добавить визуализацию в `ChartGenerator`

### Добавление нового формата отчета
1. Добавить метод в `ReportGenerator`
2. Использовать `self.results` для доступа к данным

## Обработка ошибок

### Root Cause подход
- При ошибке ищется корневая причина
- Исправляется источник проблемы, а не симптомы
- Не используются workarounds

### Примеры исправлений

#### Проблема 1: `ValueError: No group keys passed!`
**Root Cause:** `pandas.groupby()` не поддерживает пустой список группировки
**Решение:** Добавлена проверка и использование прямой агрегации для пустого `group_by`

#### Проблема 2: `UnicodeEncodeError` в Windows PowerShell
**Root Cause:** PowerShell использует кодировку cp1251, не поддерживающую Unicode emoji
**Решение:** Замена Unicode символов на ASCII-совместимые текстовые обозначения
