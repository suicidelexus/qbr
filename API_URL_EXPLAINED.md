# ❓ Что такое API URL и нужен ли он?

## TL;DR

**API URL обычно НЕ НУЖЕН!** 

В 90% случаев достаточно только **API Key**.

---

## 📖 Простое объяснение

### Что такое API URL?

**API URL** - это адрес сервера, куда отправляются запросы за данными.

**Аналогия:**
- **API Key** = ваш ключ от дома 🔑
- **API URL** = адрес дома 🏠

Если вы всегда ходите в один и тот же дом (стандартная DSP платформа), адрес уже известен. Нужен только ключ!

---

## 🎯 Когда что нужно?

### ✅ Всегда нужен: API Key

Это ваш личный ключ для доступа к данным.

**Где получить:**
1. Войдите в DSP платформу
2. Settings → API Keys
3. Generate New Key
4. Скопируйте ключ

### ⚙️ Редко нужен: API URL

Меняйте только если:
- ❌ Используете не стандартную платформу
- ❌ У вас кастомный сервер
- ❌ В документации явно указан другой URL

**По умолчанию используется:** `https://api.dsp-platform.com/v1`

### 🔧 Очень редко: Client ID

Нужен только для multi-tenant платформ (когда один аккаунт управляет несколькими клиентами).

## 🚀 Быстрая настройка

### Вариант 1: Через Dashboard (рекомендуется)

```
1. Откройте: streamlit run dashboard.py
2. Раскройте: "🔑 API Credentials"
3. Вставьте только API Key ⭐
4. Нажмите: "💾 Сохранить"

```

**Не трогайте** "⚙️ Дополнительно" - там настройки для специальных случаев.

### Вариант 2: Через .env файл

```bash
# Создайте .env файл
cp .env.example .env

# Откройте и вставьте ТОЛЬКО API Key
nano .env
```

```env
# Достаточно только этого:
DSP_API_KEY=ваш_настоящий_ключ_здесь

# Остальное закомментировано и не обязательно
```

## 💡 Примеры

### Пример 1: Стандартная платформа (99% случаев)

**Нужно:**
```
API Key: sk_live_123abc456def...
```

**НЕ нужно:**
- ❌ API URL (используется дефолтный)
- ❌ Client ID

**Готово!** Просто вставьте API Key и все работает.

---

### Пример 2: Кастомная платформа (редко)

**Нужно:**
```
API Key: sk_live_123abc456def...
API URL: https://custom-dsp.mycompany.com/api/v2
```

**Когда:** Если в документации вашей платформы явно указан другой URL.

---

### Пример 3: Multi-tenant (очень редко)

**Нужно:**
```
API Key: sk_live_123abc456def...
Client ID: client_789xyz
```

**Когда:** Если вы агентство, управляющее несколькими клиентами через один аккаунт.

## 🔍 Как узнать, нужен ли мне API URL?

### Проверьте документацию вашей DSP платформы:

**✅ НЕ нужен, если:**
- Документация упоминает только "API Key"
- Нет специального раздела про "API Endpoint"
- Используете стандартную DSP платформу

**❌ Нужен, если:**
- Документация явно указывает URL типа: `https://api.ваша-платформа.com`
- Есть раздел "API Endpoint Configuration"
- У вас on-premise или custom решение

## 🆘 FAQ

### Q: У меня ошибка "401 Unauthorized"
**A:** Проблема в API Key (не в URL). Проверьте, что ключ скопирован полностью.

### Q: У меня ошибка "404 Not Found"
**A:** Возможно нужен другой API URL. Проверьте документацию платформы.

### Q: Где взять API URL?
**A:** Если он действительно нужен - смотрите в документации вашей DSP платформы в разделе "API" или "Developers".

### Q: Можно ли оставить API URL пустым?
**A:** Нет, но можно не менять дефолтное значение. Оно подходит для большинства платформ.

### Q: Что делать, если не уверен?
**A:** Просто введите API Key и нажмите "🧪 Тест". Если работает - значит API URL правильный!
